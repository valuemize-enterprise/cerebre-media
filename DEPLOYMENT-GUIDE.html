Before you start — 3 things to prepare

Get these ready before creating any accounts — it will save you time

1

Get your Anthropic API key

This powers the AI analysis. Without it, Cerebre cannot generate reports.

ℹ New Anthropic accounts receive free credits — enough to run dozens of analyses before you pay anything.
  1. Go to console.anthropic.com
  2. Click Sign up and create an account
  3. After logging in, click API Keys in the left menu
  4. Click Create Key — name it cerebre-production
  5. Copy the key immediately — it starts with sk-ant-api03-...
⚠ The key is only shown once. Save it in a text file on your computer right now.
2

Create a GitHub account and upload your code

Railway and Vercel deploy directly from GitHub — you need the code there first.

  1. Go to github.com → Sign up (free)
  2. Click the + icon at the top right → New repository
  3. Name it cerebre-media → Leave everything else as default → Create repository
  4. On the next page, click uploading an existing file
  5. Drag your entire cerebre-media project folder into the upload area
  6. Scroll down, click Commit changes
✓ When you can see all your project files listed on github.com/yourusername/cerebre-media — you are ready.
3

Open a text file to save credentials as you go

You will collect 12 values across the next steps. Keep them in one place.

Open Notepad (Windows) or TextEdit (Mac) and paste this template — fill it in as you go:

# ─── MY CEREBRE CREDENTIALS ─────────────────────────
# Fill these in as you complete each step

ANTHROPIC_API_KEY=sk-ant-api03-...        # Step 5
DATABASE_URL=postgresql://...              # Step 2
REDIS_URL=rediss://...                     # Step 3
AWS_ACCESS_KEY_ID=...                      # Step 4
AWS_SECRET_ACCESS_KEY=...                  # Step 4
S3_ENDPOINT=https://....r2.cloudflarestorage.com # Step 4
JWT_SECRET=make-up-a-long-random-password  # You decide
RAILWAY_API_URL=https://....up.railway.app  # Step 6
VERCEL_URL=https://....vercel.app           # Step 8
1

GitHub — your code is already here

If you uploaded your code in Prerequisite Step 2, you can skip this section

✓ If you completed Prerequisite Step 2 and can see your files at github.com/[your-username]/cerebre-media, you are done with this step. Move to Step 2.

Supabase — create your database

Where all your data lives: users, reports, metrics, goals, analysis results — estimated time: 10 minutes

1

Sign up at supabase.com

Go to supabase.com → click Start your project → sign up with your GitHub account (easiest option)

2

Create a new project

Click New project. Fill in the form exactly as shown below.

FieldWhat to enter
Namecerebre-media
Database PasswordCreate a strong password — mix of letters, numbers, symbols. Save it to your credentials file now.
RegionChoose West EU (Ireland) or East US (N. Virginia) — whichever is closer to Nigeria

Click Create new project. Wait 2–3 minutes for the green "Project is ready" message.

3

Get your DATABASE_URL

This is the connection string your app will use to talk to the database.

  1. In your Supabase project, click Settings (gear icon, bottom-left)
  2. Click Database in the left menu
  3. Scroll down to the section called "Connection string"
  4. Click the URI tab
  5. Click the copy icon to copy the full connection string
ℹ The URL will look like this — the [YOUR-PASSWORD] part will actually be the password you set in Step 2: 
postgresql://postgres:[YOUR-PASSWORD]@db.abcdefgh.supabase.co:5432/postgres

# Replace [YOUR-PASSWORD] with the actual password you created.
# Save this full URL as DATABASE_URL in your credentials file.
4

Run the database schema

This creates all the tables (users, reports, metrics, etc.) in your new database.

  1. In Supabase, click SQL Editor in the left menu
  2. Click the New query button
  3. On your computer, open the file cerebre-media/backend/src/db/schema.sql
  4. Select ALL the text in that file (Ctrl+A or Cmd+A) and copy it
  5. Paste it into the Supabase SQL editor
  6. Click the green Run button
✓ You should see "Success. No rows returned" — this means all tables were created. You are done with Supabase.

Upstash — create your Redis job queue

This manages the queue of files waiting to be processed by the AI — estimated time: 5 minutes

1

Sign up at upstash.com

Go to upstash.com → click Sign up → use Continue with GitHub for speed

2

Create a Redis database

  1. After logging in, click Create database
  2. Name: cerebre-queue
  3. Type: Regional
  4. Region: Choose closest to Nigeria — either EU-West-1 or US-East-1
  5. Click Create
3

Get your REDIS_URL

After the database is created, you need the connection URL.

  1. Click on your new database name
  2. Scroll down to the Details section
  3. Find the field called Endpoint and note the URL
  4. Find the field called Password and click the eye icon to reveal it

Your REDIS_URL is built like this:

# Your REDIS_URL format:
REDIS_URL=rediss://default:[PASSWORD]@[ENDPOINT]:6379

# Example (yours will have different values):
rediss://default:AbCdEfGhIj123@pleased-porpoise-12345.upstash.io:6379

# Save this as REDIS_URL in your credentials file.
⚠ Note the rediss:// prefix (with double s) — this means secure Redis with TLS encryption. This is correct.

Cloudflare R2 — file storage

Stores the uploaded PDF and image files — 10GB free — estimated time: 8 minutes

1

Sign up at cloudflare.com

Go to cloudflare.com → click Sign up → create a free account

2

Enable R2 storage

  1. In the Cloudflare dashboard, click R2 in the left sidebar
  2. You will be asked to add a payment method to enable R2. You will not be charged unless you exceed 10GB of storage or 10 million requests per month — this application will not come close to either limit when starting out
  3. Add a card and enable R2
3

Create a bucket

A bucket is like a folder that holds all the uploaded files.

  1. Click Create bucket
  2. Bucket name: cerebre-media-files
  3. Leave all other settings as default
  4. Click Create bucket
4

Get your Account ID and create API tokens

This is the most important step for file storage — you need 3 values.

First: get your Account ID

  1. Look at the URL in your browser — it will be:
    dash.cloudflare.com/[ACCOUNT-ID]/r2/...
  2. Copy the Account ID (long string of letters and numbers)
  3. Your S3_ENDPOINT is: https://[ACCOUNT-ID].r2.cloudflarestorage.com

Then: create API tokens

  1. Click Manage R2 API tokens (top-right area on the R2 page)
  2. Click Create API token
  3. Name: cerebre-backend
  4. Under Permissions, select Object Read and Write
  5. Click Create API Token
⚠ The Secret Access Key is shown ONCE. Copy it immediately. If you miss it, you have to delete the token and create a new one. 
# Save these 3 values to your credentials file:
AWS_ACCESS_KEY_ID     = the "Access Key ID" shown on the page
AWS_SECRET_ACCESS_KEY = the "Secret Access Key" shown (ONCE only)
S3_ENDPOINT           = https://[YOUR-ACCOUNT-ID].r2.cloudflarestorage.com
S3_BUCKET_NAME        = cerebre-media-files
AWS_REGION            = auto

Something went wrong? Click your problem

These are the 10 most common deployment issues — click each one to see the fix

🔴

The website loads but shows "Network Error" or "Failed to connect"

▶

This means the frontend cannot reach the backend API. Check these in order:

  1. Go to Vercel → your project → Settings → Environment Variables
  2. Confirm NEXT_PUBLIC_API_URL is set and ends with /api
  3. Confirm the Railway URL in that variable is correct — test it by visiting [railway-url]/health directly in your browser
  4. If you changed the variable, go to Vercel → Deployments → click Redeploy on the latest deployment
🔴

Login works but returns "Invalid token" or I get logged out immediately

▶

Your JWT_SECRET is either missing or too short. Go to Railway → API service → Variables and confirm JWT_SECRET is set and is at least 32 characters long. After updating it, Railway will automatically redeploy.

🔴

Vercel build fails — "Module not found" or TypeScript errors

▶

The most common cause is the Root Directory not being set correctly.

  1. Go to Vercel → your project → Settings → General
  2. Find Root Directory and confirm it says frontend
  3. If it's blank or wrong, change it to frontend and save
  4. Go to Deployments → Redeploy
🔴

Railway build fails with a "sharp" or "tesseract" error

▶

These are native modules that need system build tools. Fix:

  1. Go to Railway → API service → Settings
  2. Find Build Command and change it to:
npm install --ignore-scripts && npm run build 2>/dev/null; echo done
🔴

File uploads fail — files disappear or show an error

▶

Almost always a Cloudflare R2 credential issue. Check all 5 of these in Railway → Variables:

  • AWS_ACCESS_KEY_ID — must match the Access Key ID from Cloudflare R2
  • AWS_SECRET_ACCESS_KEY — must match the Secret Access Key (check for typos)
  • S3_BUCKET_NAME — must exactly match your bucket name: cerebre-media-files
  • S3_ENDPOINT — must be https://[account-id].r2.cloudflarestorage.com (include https://)
  • AWS_REGION — must be set to auto for Cloudflare R2
🔴

Files upload but AI analysis never completes / stays "processing"

▶

Usually one of these three problems:

  1. Worker not running: Check Railway → your project — confirm you see TWO services and both show Active (green dot)
  2. Wrong Anthropic key: In Railway → Worker service → Variables, confirm ANTHROPIC_API_KEY starts with sk-ant-api03- and has no extra spaces
  3. Redis not connecting: Confirm REDIS_URL in BOTH services starts with rediss:// (double s) and the password is correct
🔴

CORS error — browser console shows "blocked by CORS policy"

▶

The backend does not recognise your frontend's URL.

  1. Go to Railway → API service → Variables
  2. Confirm FRONTEND_URL matches your Vercel URL exactly
  3. It must include https:// and have no trailing slash
  4. Correct: https://cerebre-media.vercel.app
  5. Wrong: https://cerebre-media.vercel.app/ (trailing slash)
  6. After fixing, Railway will redeploy automatically
🔴

Supabase "database connection failed" in Railway logs

▶

The DATABASE_URL has an error. Common issues:

  • You forgot to replace [YOUR-PASSWORD] with your actual Supabase database password
  • The password contains special characters like @ or # that need to be URL-encoded — if your password has @ change it to %40 in the URL
  • The URL is missing the port — it should end in :5432/postgres

Easiest fix: go to Supabase → Settings → Database → copy the connection string again fresh and paste it carefully.

🔴

The website shows a blank white page

▶

Right-click the page → Inspect → Console tab. Look for the red error message. Most common causes:

  1. If you see "NEXT_PUBLIC_API_URL is not defined" — the environment variable wasn't saved. Add it in Vercel and redeploy.
  2. If you see "Module not found" — the Root Directory wasn't set to frontend in Vercel settings.
  3. If there is no error and just a blank page — try a hard refresh: Ctrl+Shift+R (Windows) or Cmd+Shift+R (Mac)
🟡

Railway shows a "sleeping" or "inactive" service

▶

Railway's free tier sleeps services after periods of inactivity. The first request after a sleep takes 20–30 seconds to wake up — this is normal on the free tier. To avoid this, upgrade to Railway's Starter plan ($5/month) which keeps services always on.

What to do after deployment

Your app is live — here is what to do in the first hour

1

Create your real admin account

Don't use the demo account — create a proper account for your team.

  1. On the login page, click Register
  2. Create an account with your real email
  3. Then go to Supabase → SQL Editor → run this to make yourself admin:
UPDATE users SET role = 'admin' WHERE email = 'your@email.com';
-- Replace with the email you just registered with
2

Upload your first real report

Test the full pipeline with a real platform PDF.

  1. Download an Insights export from Instagram, Facebook, Google Ads, or any platform
  2. Go to Upload Reports in Cerebre
  3. Drag the PDF or image file onto the upload area
  4. Watch it process — it should take 30–60 seconds
  5. When complete, click View Report to see the full AI analysis
3

Set your first business goal

The AI tailors all recommendations to your active goals.

  1. Click Goals in the sidebar
  2. Click Add goal
  3. Choose a category, set a target, and pick a deadline
  4. The AI will generate a strategy breakdown immediately
4

Add a custom domain (optional but recommended)

To use your own domain like analytics.yourcompany.com instead of the default Vercel URL:

  1. Go to Vercel → your project → Settings → Domains
  2. Add your custom domain
  3. Follow Vercel's instructions to update your DNS records (this takes 10–30 minutes to propagate)
  4. Vercel handles SSL automatically — no extra configuration needed
5

Invite your team

Share the website URL with your team. They can click Register on the login page to create their own accounts. As admin, you can manage their access in the Admin Panel (shield icon in the sidebar).
